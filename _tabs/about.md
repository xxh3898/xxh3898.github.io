`xxh3898`는 Java, Spring, Database, Infra 문제 해결 과정을 기록하는 기술 블로그입니다.

이 블로그의 목적은 공부한 내용을 나열하는 것이 아니라, 실제 프로젝트에서 마주친 문제를 어떻게 분석하고 해결했는지 남기는 것입니다. 공개 포트폴리오에서는 결과를 요약하고, 이 블로그에서는 판단 근거와 검증 과정을 더 자세히 정리합니다.

## 운영 원칙

- 문제 상황, 원인 분석, 해결 과정, 검증 결과를 한 글 안에서 함께 다룹니다.
- 회사 내부정보, secret, 실제 운영 로그, 고객사 정보는 공개하지 않습니다.
- PKM에 남긴 개인 기록은 공개 가능한 수준으로 일반화한 뒤 글로 발행합니다.
- 기존 Tistory 글은 유지하고, 새 기술 글은 GitHub Pages를 중심으로 정리합니다.
- Tistory에는 필요할 때 요약본이나 원문 링크를 함께 올려 유입 경로로 활용합니다.

## 주요 주제

- `Spring Security`, `JWT`, 인증/인가 흐름
- `JPA`, `QueryDSL`, N+1, fetch join, 실행 계획
- `MySQL`, `Redis`, 캐시와 읽기 모델 분리
- `Docker`, `Nginx`, `AWS`, 배포와 운영 검증
- `JUnit 5`, `MockMvc`, `Testcontainers`, `Spring REST Docs`
